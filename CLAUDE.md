## What This Project Is

OmWhisper is a cross-platform (macOS + Windows) menu-bar app for real-time, on-device speech transcription.
Built with Tauri 2 (Rust backend) + React + TypeScript frontend.
Uses OpenAI Whisper (via whisper-rs / whisper.cpp) running entirely locally.
No audio ever leaves the device.

- **macOS**: Metal GPU-accelerated transcription, Push-to-Talk (CGEventTap), built-in LLM, auto-paste via Accessibility API
- **Windows**: CPU-only transcription, toggle-mode only, auto-paste via SendInput, NSIS installer built by GitHub Actions CI

**App identifier:** `com.omwhisper.app`
**Global hotkey:** `Cmd+Shift+V` (toggle recording)
**Theme:** Emerald/teal (`#34d399`, `#6ee7b7`, `#2dd4bf`) on deep green-black (`#0a0f0d`)
**Logo:** ॐ symbol in emerald gradient

## Key Design Decisions

- **Model path resolution**: Rust resolves relative model paths by walking up 4 levels from the binary (`debug/ → target/ → src-tauri/ → project root`). Works in dev; production will use `appDataDir`.
- **Window hidden by default**: App lives in menu bar. `visible: false` in `tauri.conf.json`. Show via tray click or hotkey.
- **Audio pipeline**: cpal stream runs inside a background thread (non-Send workaround). Sends 3-second f32 chunks via `mpsc::channel` to WhisperEngine thread.
- **Global shortcut**: Emits `hotkey-toggle-recording` event to frontend to start; directly stops capture and emits `recording-state: false` to stop.
- **Icons**: Generated from `landing/omwhisper-logo.svg` using `rsvg-convert` + `iconutil`. Run regeneration script if SVG changes.
- **whisper-rs Metal GPU split**: `whisper-rs = { version = "0.14" }` (CPU-only) in `[dependencies]`; `whisper-rs = { version = "0.14", features = ["metal"] }` in `[target.'cfg(target_os = "macos")'.dependencies]`. Cargo unions both on macOS → Metal GPU unchanged. Windows/Linux → CPU only. Do NOT remove the base entry or Windows breaks.
- **PTT gating**: Single-key PTT (Fn, CapsLock, Right Option/Control) via `fn_key.rs` CGEventTap is `#[cfg(target_os = "macos")]` only. Plugin-based PTT shortcut is `#[cfg(not(target_os = "windows"))]`. PTT UI section hidden on Windows.
- **LlmEngine (built-in LLM)**: `llama-cpp-2` (Metal) and all `LlmEngine` wiring are `#[cfg(target_os = "macos")]`. Built-in backend option hidden in UI on non-macOS via `get_platform` command.
- **Windows paste**: `paste.rs` uses `SendInput` (Win32 API) on Windows to simulate Cmd+V. macOS uses `CGEventPost`. Both paths live in the same file, gated by `#[cfg]`.
- **Windows build**: GitHub Actions (`build-windows.yml`) builds the NSIS installer on `windows-latest`. Triggers on `v*` tag push and manual dispatch. macOS builds remain local via `scripts/build-release.sh`.

## Notes & Technical Decisions

### Prompt 5 — VAD Approach Decision
**Chose: Option B (Energy-based RMS VAD)**
**Reason:** Faster to ship, no additional model dependency. Gets the job done for v0.1.
**Future upgrade (Option A — Silero VAD):**
- Silero VAD is a tiny ONNX model (~1.8MB) that detects speech with high accuracy
- Would need `ort` crate (ONNX Runtime bindings for Rust)
- Download `silero_vad.onnx` on first launch alongside Whisper model
- Process audio in 512-sample frames, get speech probability per frame
- Gate Whisper inference only on frames with probability > 0.5
- Eliminates hallucinations on silence much more reliably than RMS threshold
- **Revisit after Phase 2 is complete**

### Whisper Hallucinations (Current Known Issue)
Whisper hallucinates text when fed silence (e.g., "Thank you for watching.", "And so...").
Energy-based VAD (Prompt 5) will fix this by only sending audio above RMS threshold.

### Model Storage (Dev vs Prod)
- **Dev**: models at `{project_root}/models/ggml-*.bin` — resolved by walking 4 levels up from the debug binary
- **Prod / .app bundle**: models at `~/Library/Application Support/com.omwhisper.app/models/`
- `resolve_model_path` checks the app data dir first, then falls back to the dev walk-up path
- Copy dev models to app data dir for bundle testing: `cp models/*.bin ~/Library/Application\ Support/com.omwhisper.app/models/`

### Auto-Paste & Accessibility (macOS TCC)
- Auto-paste uses `CGEventPost` (CoreGraphics) to send Cmd+V to the target app
- Requires **Accessibility permission** for the OmWhisper process (`AXIsProcessTrusted()`)
- **Dev pain**: every `cargo build` changes the binary's ad-hoc code signature hash → macOS revokes TCC entry
- **Fix**: after each `cargo build`, re-sign with a stable identifier:
  ```bash
  codesign --force --sign - --identifier "com.omwhisper.app" target/debug/omwhisper
  ```
- For the `.app` bundle: `codesign --force --deep --sign - --identifier "com.omwhisper.app" target/debug/bundle/macos/OmWhisper.app`
- **Production** (properly signed `.app`): Accessibility survives reinstalls automatically — no manual re-signing needed
- User must grant Accessibility **after** the binary is signed with the stable identifier, **before** testing paste

## Known Issues / Tech Debt

- [ ] WhisperEngine reloads model on every recording session → should be cached in state
- [ ] Windows: no PTT support (toggle-only) — would require a Windows-native low-level hook equivalent to CGEventTap
- [ ] Windows: no built-in LLM (llama-cpp-2 Metal is macOS-only) — would need a CPU/CUDA llama.cpp backend
- [ ] Windows: Accessibility permission model is different — auto-paste via SendInput works without special permissions but may be blocked by some apps
